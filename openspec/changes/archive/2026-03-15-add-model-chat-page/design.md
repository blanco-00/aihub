## Context

当前系统已实现模型管理功能，用户可以在 `/system/model` 页面配置各种AI模型（OpenAI、智谱、阿里等）。但缺乏一个面向用户的聊天入口。

现有代码：
- **后端 API**: `ModelChatController` 提供 `/api/ai/chat` 接口，支持单轮对话
- **前端测试页面**: `frontend/src/views/system/model/test/index.vue` - 这是一个测试工具页面，UI 较简陋，不适合普通用户使用
- **前端 API**: `chatWithModel` 函数已封装

## Goals / Non-Goals

**Goals:**
1. 创建美观的AI聊天页面，支持模型选择和多轮对话
2. 页面入口位于 `/ai/chat`，通过侧边栏菜单访问
3. 复用现有 `/api/ai/chat` 接口，无需修改后端
4. 参考现有测试页面的功能逻辑，但提升UI体验

**Non-Goals:**
- 对话历史持久化存储（后续迭代）
- 流式响应（后续迭代，当前API为同步）
- 多模态输入（语音、图片等）

## Decisions

### 1. 页面位置

- **决策**: 新建 `frontend/src/views/ai/chat/index.vue`
- **理由**: 
  - 遵循现有项目结构，`ai` 目录下放置AI相关页面
  - 与现有测试页面 `/system/model/test` 分开，职责清晰

### 2. UI 风格

- **决策**: 采用类似 ChatGPT 的现代简洁风格
- **理由**:
  - 用户对 ChatGPT 风格熟悉，上手成本低
  - 现有 `chatai/components` 目录下有多种风格组件可参考
  - 使用 Element Plus 组件确保一致性

### 3. 消息展示

- **决策**: 用户消息右对齐，模型消息左对齐
- **理由**: 符合主流聊天软件习惯，用户消息更醒目

### 4. 模型选择

- **决策**: 页面顶部固定模型下拉选择器
- **理由**:
  - 切换模型方便
  - 参考现有测试页面，用户反馈良好

## Risks / Trade-offs

- **[风险] 页面加载时模型列表为空**
  - **缓解**: 页面加载时自动获取已启用的模型列表，如果没有可用模型，提示用户先去配置
- **[风险] API 请求失败**
  - **缓解**: 统一错误处理，展示友好的错误提示
- **[风险] 对话内容丢失**
  - **缓解**: 当前仅支持页面内存存储，刷新后清空（持久化在后续迭代）
