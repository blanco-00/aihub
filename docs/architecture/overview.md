## 🔑 核心组件

### 1. Agents网关
- **功能**：统一Agent入口，路由转发、负载均衡、流量控制
- **详细设计**：见 [网关架构设计](./gateways.md#agents网关)

### 2. 模型网关
- **功能**：统一模型接入，流量分发、Token统计、模型切换
- **详细设计**：见 [网关架构设计](./gateways.md#模型网关)

### 3. MCP网关
- **功能**：MCP工具统一管理，连接池、流量控制
- **详细设计**：见 [网关架构设计](./gateways.md#mcp网关)

### 4. Agent模板系统
- **功能**：可复用的Agent模板，快速创建Agent实例
- **详细设计**：见 [Agent模板方案](../agent-template-solution.md)

## 🔄 数据流

### Agent执行流程
```
用户请求
  ↓
Agents网关 (路由、负载均衡)
  ↓
Agent服务 (基于模板实例)
  ↓
决策引擎
  ├─→ RAG检索 (从RAG库)
  ├─→ MCP工具调用 (通过MCP网关)
  └─→ LLM推理 (通过模型网关)
  ↓
结果融合
  ↓
返回用户
```

### 模型调用流程
```
Agent服务请求
  ↓
模型网关 (路由选择、负载均衡)
  ↓
模型管理服务 (获取模型配置)
  ↓
调用外部API (OpenAI/Claude等)
  ↓
Token统计、监控
  ↓
返回结果
```

## 🛠️ 技术栈

### 前端
- **框架**: Vue 3 + TypeScript
- **UI库**: Element Plus
- **构建**: Vite
- **状态管理**: Pinia
- **路由**: Vue Router

### 后端
- **主框架**: Spring Boot 3.x (Java 17+)
- **AI框架**: Spring AI
- **数据库**: MySQL 8.0+
- **缓存**: Redis
- **认证**: Spring Security + JWT

### 网关
- **实现方式**: Spring Cloud Gateway 或 Spring Boot内嵌
- **负载均衡**: 内置或集成Nginx
- **监控**: Spring Boot Actuator + Micrometer

## 📊 核心特性

1. **统一网关入口** - 所有请求通过网关统一管理
2. **模板化Agent** - 快速创建Agent实例
3. **资源统一管理** - Prompt、RAG、MCP统一管理
4. **流量控制** - 支持限流、熔断、降级
5. **监控统计** - Token消耗、性能监控、告警
6. **权限管理** - 三级权限控制（人-角色-资源）

## 🎯 设计原则

1. **网关化** - 所有外部请求通过网关
2. **模板化** - Agent、Prompt等可复用模板
3. **可扩展** - 支持水平扩展
4. **可观测** - 完善的监控和日志
5. **安全性** - 统一的权限控制和审计
