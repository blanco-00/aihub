## 初始化流程

### 1. 数据库初始化

数据库初始化有两种方式，详细说明请参考 [SQL 脚本管理文档](../sql/guide.md)：

#### 方式一：手动执行 SQL 脚本（推荐）

直接执行全量初始化脚本：

```bash
mysql -u root -p < docs/sql/init/init_v1.0.0.sql
```

**脚本位置**: `docs/sql/init/init_v1.0.0.sql`

#### 方式二：使用 Flyway（可选，仅开发环境）

如果项目引入了 Flyway，可以在开发环境自动执行迁移脚本。

**配置说明**: 详细配置请参考 [SQL 脚本管理文档 - Flyway 集成](../sql/guide.md#-flyway-集成可选仅开发环境)

**重要原则**: Flyway 只在开发环境启用，测试和生产环境必须禁用。

### 2. 超级管理员初始化

**重要**: 系统不会自动创建超级管理员，需要通过初始化页面手动创建。

#### 初始化页面

系统启动后，如果检测到没有超级管理员，会自动跳转到初始化页面（`/init`），引导用户创建第一个超级管理员。

#### 初始化流程

1. 执行数据库初始化脚本（创建表结构）
2. 启动应用
3. 访问系统，自动跳转到初始化页面
4. 填写超级管理员信息（用户名、邮箱、密码）
5. 提交创建，完成初始化

#### 初始化页面功能

- 检查系统是否已初始化（是否存在超级管理员）
- 如果未初始化，显示创建超级管理员表单
- 如果已初始化，跳转到登录页面
- 表单验证（用户名、邮箱、密码格式）
- 密码强度提示

## Java 初始化服务实现

### 1. 初始化服务接口

**核心方法**:
- `isInitialized()`: 检查系统是否已初始化（是否存在超级管理员）
- `createSuperAdmin(dto)`: 创建超级管理员（仅在没有超级管理员时可用）

### 2. 初始化服务实现逻辑

**isInitialized() 逻辑**:
```
查询数据库中是否存在 role = 'SUPER_ADMIN' 且 is_deleted = 0 的用户
返回: true-已初始化, false-未初始化
```

**createSuperAdmin() 逻辑**:
```
1. 检查是否已存在超级管理员 -> 如果存在，抛出异常
2. 检查用户名是否已存在 -> 如果存在，抛出异常
3. 检查邮箱是否已存在 -> 如果存在，抛出异常
4. 使用 BCrypt 加密密码
5. 创建用户，设置 role = 'SUPER_ADMIN'
6. 保存到数据库
```

### 3. 初始化控制器

**API 接口**:
- `GET /api/init/status`: 检查系统是否已初始化
- `POST /api/init/super-admin`: 创建超级管理员

**请求/响应**:
- 创建请求: `{ username, email, password }`
- 响应: 统一结果格式 `Result<T>`

### 4. 初始化DTO

**字段**:
- `username`: 用户名（必填，3-50个字符）
- `email`: 邮箱（必填，邮箱格式验证）
- `password`: 密码（必填，8-50个字符）

### 5. 前端路由拦截

**拦截逻辑**:
```
1. 排除初始化相关路径（/api/init, /init, 静态资源）
2. 检查系统是否已初始化
3. 如果未初始化，跳转到 /init 页面
4. 如果已初始化，正常访问
```

## 超级管理员保护

### 删除保护

在用户删除逻辑中，需要检查是否为最后一个超级管理员：

**删除逻辑**:
```
1. 查询用户信息
2. 如果用户角色是 SUPER_ADMIN:
   - 查询当前超级管理员总数
   - 如果总数 <= 1，抛出异常（至少保留一个）
3. 执行逻辑删除
```

### 角色修改保护

在修改用户角色时，也需要保护最后一个超级管理员：

**角色修改逻辑**:
```
1. 查询用户信息
2. 如果从 SUPER_ADMIN 改为其他角色:
   - 查询当前超级管理员总数
   - 如果总数 <= 1，抛出异常（至少保留一个）
3. 更新用户角色
```

## 用户角色枚举

建议定义用户角色枚举，包含以下角色：

- `SUPER_ADMIN`: 超级管理员
- `ADMIN`: 管理员
- `USER`: 普通用户

## 前端初始化页面设计

### 页面路径
- 初始化页面: `/init`
- API接口: `/api/init/*`

### 页面功能

1. **初始化状态检查**
   - 页面加载时，调用 `GET /api/init/status` 检查系统是否已初始化
   - 如果已初始化，自动跳转到登录页面
   - 如果未初始化，显示创建超级管理员表单

2. **创建超级管理员表单**
   - 用户名输入框（必填，3-50个字符）
   - 邮箱输入框（必填，邮箱格式验证）
   - 密码输入框（必填，8-50个字符，显示密码强度）
   - 确认密码输入框（必填，与密码一致验证）
   - 提交按钮

3. **表单验证**
   - 实时验证输入格式
   - 密码强度指示器（弱/中/强）
   - 提交前完整验证
   - 错误提示清晰明确

4. **提交处理**
   - 调用 `POST /api/init/super-admin` 创建超级管理员
   - 显示加载状态
   - 成功后显示成功提示
   - 3秒后自动跳转到登录页面

### 页面设计要点

- **科技感设计**: 体现AI项目的科技属性
- **简洁明了**: 表单布局清晰，操作简单
- **友好提示**: 清晰的说明文字和错误提示
- **响应式**: 支持不同屏幕尺寸

### Vue 3 组件实现要点

**组件结构**:
- 使用 Element Plus 的 `el-form` 组件
- 表单字段: 用户名、邮箱、密码、确认密码
- 密码强度指示器（弱/中/强）

**核心逻辑**:
```
1. 页面加载时:
   - 调用 GET /api/init/status 检查初始化状态
   - 如果已初始化，跳转到 /login
   - 如果未初始化，显示表单

2. 表单验证:
   - 用户名: 必填，3-50个字符
   - 邮箱: 必填，邮箱格式验证
   - 密码: 必填，8-50个字符
   - 确认密码: 必填，与密码一致

3. 密码强度计算:
   - 根据长度、大小写、数字、特殊字符计算强度
   - 显示: 弱/中/强

4. 提交处理:
   - 验证表单
   - 调用 POST /api/init/super-admin
   - 显示加载状态
   - 成功后提示，3秒后跳转到登录页
```
