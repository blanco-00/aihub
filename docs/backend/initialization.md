## 初始化流程

> **前置步骤**: 在开始初始化之前，请确保已完成：
> 1. 创建数据库（参考 [快速开始指南](./quick-start.md#1-创建数据库)）
> 2. 启动前后端服务（参考 [快速开始指南](./quick-start.md#3-启动前后端服务)）

### 1. 数据库配置（页面引导）

**重要**: 系统提供了页面引导的数据库配置功能，无需手动编辑配置文件。

> **详细步骤请参考** [快速开始指南 - 配置数据库连接](./quick-start.md#4-配置数据库连接页面引导推荐)

#### 简要流程

1. 访问 `http://localhost:3000`，系统会自动跳转到 `/setup` 页面
2. 填写数据库信息（主机、端口、数据库名、用户名、密码）
3. 点击"测试连接"验证配置
4. 点击"保存配置"保存到 `application-local.yml`
5. **重启后端服务**使配置生效

**配置说明**：
- 配置信息保存在 `backend/aihub-api/src/main/resources/application-local.yml`
- 该文件已在 `.gitignore` 中，不会被提交到 Git
- 支持空密码（如果 MySQL 用户没有密码）

### 2. 数据库表结构初始化

> **详细步骤请参考** [快速开始指南 - 初始化数据库表结构](./quick-start.md#5-初始化数据库表结构通过页面)

#### 方式一：通过页面初始化（推荐）

1. 完成数据库配置并重启后端服务
2. 访问 `http://localhost:3000`，系统会自动跳转到 `/init` 页面
3. 如果数据库连接正常但表未初始化，会显示"初始化数据库表结构"按钮
4. 点击按钮，系统会自动执行 SQL 脚本创建所有表结构

**脚本位置**: `backend/aihub-api/src/main/resources/db/migration/V1.0.0__init_tables.sql`

#### 方式二：手动执行 SQL 脚本

如果需要手动执行，可以使用全量脚本：

```bash
mysql -u root -p < docs/sql/init/init_v1.0.0.sql
```

**全量脚本位置**: `docs/sql/init/init_v1.0.0.sql`（包含 CREATE DATABASE）

#### 方式三：使用 Flyway（可选，仅开发环境）

如果项目引入了 Flyway，可以在开发环境自动执行迁移脚本。

**配置说明**: 详细配置请参考 [SQL 脚本管理文档 - Flyway 集成](../sql/guide.md#-flyway-集成可选仅开发环境)

**重要原则**: Flyway 只在开发环境启用，测试和生产环境必须禁用。

### 3. 超级管理员初始化

**重要**: 系统不会自动创建超级管理员，需要通过初始化页面手动创建。

> **详细步骤请参考** [快速开始指南 - 创建超级管理员](./quick-start.md#6-创建超级管理员)

#### 初始化流程

1. 完成数据库配置和表结构初始化
2. 访问系统，自动跳转到初始化页面（`/init`）
3. 填写超级管理员信息：
   - 用户名：3-50个字符，不能为空
   - 邮箱：有效的邮箱地址
   - 密码：8-50个字符，建议包含大小写字母、数字和特殊字符
   - 确认密码：与密码一致
4. 提交创建，完成初始化
5. 自动跳转到首页

#### 初始化页面功能

- **数据库状态检查**：自动检查数据库连接状态和表结构初始化状态
- **数据库表结构初始化**：如果表未初始化，提供一键初始化功能
- **初始化状态检查**：检查系统是否已初始化（是否存在超级管理员）
- **创建超级管理员表单**：如果未初始化，显示创建表单
- **自动跳转**：如果已初始化，自动跳转到首页
- **表单验证**：用户名、邮箱、密码格式验证
- **密码强度提示**：实时显示密码强度
- **防重复提交**：按钮加载状态，防止重复点击

## Java 初始化服务实现

### 1. 初始化服务接口

**核心方法**:
- `isInitialized()`: 检查系统是否已初始化（是否存在超级管理员）
- `createSuperAdmin(dto)`: 创建超级管理员（仅在没有超级管理员时可用）

### 2. 初始化服务实现逻辑

**isInitialized() 逻辑**:
```
查询数据库中是否存在 role = 'SUPER_ADMIN' 且 is_deleted = 0 的用户
返回: true-已初始化, false-未初始化
```

**createSuperAdmin() 逻辑**:
```
1. 检查是否已存在超级管理员 -> 如果存在，抛出异常
2. 检查用户名是否已存在 -> 如果存在，抛出异常
3. 检查邮箱是否已存在 -> 如果存在，抛出异常
4. 使用 BCrypt 加密密码
5. 创建用户，设置 role = 'SUPER_ADMIN'
6. 保存到数据库
```

### 3. 初始化控制器

**API 接口**:
- `GET /api/init/status`: 检查系统是否已初始化
- `POST /api/init/super-admin`: 创建超级管理员

**请求/响应**:
- 创建请求: `{ username, email, password }`
- 响应: 统一结果格式 `Result<T>`

### 4. 初始化DTO

**字段**:
- `username`: 用户名（必填，3-50个字符）
- `email`: 邮箱（必填，邮箱格式验证）
- `password`: 密码（必填，8-50个字符）

### 5. 前端路由拦截

**拦截逻辑**:
```
1. 检查数据库是否已配置
   - 如果未配置，跳转到 /setup 页面（数据库配置页面）
   - 如果已配置，继续下一步

2. 检查系统是否已初始化
   - 如果未初始化，跳转到 /init 页面（初始化页面）
   - 如果已初始化，正常访问

3. 排除路径：
   - /api/setup/** - 数据库配置 API
   - /api/init/** - 初始化 API
   - /setup - 数据库配置页面
   - /init - 初始化页面
   - /error - 错误页面
   - 静态资源路径
```

**路由守卫特性**：
- 支持网络错误处理，自动跳转到错误页面
- 支持数据库配置状态检查
- 支持系统初始化状态检查
- 防止循环跳转

## 超级管理员保护

### 删除保护

在用户删除逻辑中，需要检查是否为最后一个超级管理员：

**删除逻辑**:
```
1. 查询用户信息
2. 如果用户角色是 SUPER_ADMIN:
   - 查询当前超级管理员总数
   - 如果总数 <= 1，抛出异常（至少保留一个）
3. 执行逻辑删除
```

### 角色修改保护

在修改用户角色时，也需要保护最后一个超级管理员：

**角色修改逻辑**:
```
1. 查询用户信息
2. 如果从 SUPER_ADMIN 改为其他角色:
   - 查询当前超级管理员总数
   - 如果总数 <= 1，抛出异常（至少保留一个）
3. 更新用户角色
```

## 用户角色枚举

建议定义用户角色枚举，包含以下角色：

- `SUPER_ADMIN`: 超级管理员
- `ADMIN`: 管理员
- `USER`: 普通用户

## 前端初始化页面设计

### 页面路径
- 数据库配置页面: `/setup`
- 初始化页面: `/init`
- API接口: `/api/setup/*`, `/api/init/*`

### 数据库配置页面（/setup）

**页面功能**：
1. **数据库配置表单**：
   - 数据库主机输入框
   - 数据库端口输入框（数字输入）
   - 数据库名称输入框
   - 用户名输入框
   - 密码输入框（支持显示/隐藏）

2. **连接测试功能**：
   - "测试连接"按钮
   - 实时显示连接测试结果
   - 显示连接耗时
   - 显示数据库是否存在
   - 错误信息提示

3. **保存配置功能**：
   - "保存配置"按钮（仅在连接测试成功后启用）
   - 配置保存到 `application-local.yml`
   - 自动激活 `local` profile
   - 成功提示和"重启后继续"按钮

4. **防重复提交**：
   - 按钮加载状态
   - 防止重复点击
   - 跳转中状态提示

**页面设计要点**：
- 科技感设计，体现AI项目属性
- 清晰的表单布局和操作提示
- 友好的错误提示和帮助信息
- 响应式设计，支持不同屏幕尺寸

### 初始化页面功能（/init）

1. **数据库状态检查**
   - 页面加载时，调用 `GET /api/init/database-status` 检查数据库状态
   - 显示数据库连接状态、数据库是否存在、表结构是否已初始化
   - 如果数据库未连接或不存在，显示错误提示

2. **数据库表结构初始化**
   - 如果数据库连接正常但表未初始化，显示"初始化数据库表结构"按钮
   - 点击按钮，调用 `POST /api/init/database` 初始化表结构
   - 显示加载状态和初始化进度
   - 初始化完成后，自动刷新数据库状态

3. **初始化状态检查**
   - 数据库表结构初始化完成后，调用 `GET /api/init/status` 检查系统是否已初始化
   - 如果已初始化，自动跳转到首页
   - 如果未初始化，显示创建超级管理员表单

4. **创建超级管理员表单**
   - 用户名输入框（必填，3-50个字符）
   - 邮箱输入框（必填，邮箱格式验证）
   - 密码输入框（必填，8-50个字符，显示密码强度）
   - 确认密码输入框（必填，与密码一致验证）
   - 提交按钮

5. **表单验证**
   - 实时验证输入格式
   - 密码强度指示器（弱/中/强）
   - 提交前完整验证
   - 错误提示清晰明确

6. **提交处理**
   - 调用 `POST /api/init/super-admin` 创建超级管理员
   - 显示加载状态
   - 成功后显示成功提示
   - 自动跳转到首页

### 页面设计要点

- **科技感设计**: 体现AI项目的科技属性
- **简洁明了**: 表单布局清晰，操作简单
- **友好提示**: 清晰的说明文字和错误提示
- **响应式**: 支持不同屏幕尺寸

### Vue 3 组件实现要点

**组件结构**:
- 使用 Element Plus 的 `el-form` 组件
- 表单字段: 用户名、邮箱、密码、确认密码
- 密码强度指示器（弱/中/强）

**核心逻辑**:
```
1. 页面加载时:
   - 调用 GET /api/init/status 检查初始化状态
   - 如果已初始化，跳转到 /login
   - 如果未初始化，显示表单

2. 表单验证:
   - 用户名: 必填，3-50个字符
   - 邮箱: 必填，邮箱格式验证
   - 密码: 必填，8-50个字符
   - 确认密码: 必填，与密码一致

3. 密码强度计算:
   - 根据长度、大小写、数字、特殊字符计算强度
   - 显示: 弱/中/强

4. 提交处理:
   - 验证表单
   - 调用 POST /api/init/super-admin
   - 显示加载状态
   - 成功后提示，3秒后跳转到登录页
```
