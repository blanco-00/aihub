## 初始化步骤

### 1. 准备数据库

确保已安装 MySQL 8.0+，并创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS aihub DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 执行初始化脚本

#### 方式一：使用全量脚本（推荐用于新环境）

直接执行全量初始化脚本：

```bash
mysql -u root -p aihub < docs/sql/init/init_v1.0.0.sql
```

#### 方式二：使用增量脚本（可选，需要 Flyway）

如果项目使用了 Flyway，可以执行增量脚本：

```bash
# 复制迁移脚本到资源目录
cp docs/sql/migrations/V1.0.0__init_tables.sql src/main/resources/db/migration/

# 启动应用，Flyway 会自动执行
```

**注意**: 增量脚本不包含 `CREATE DATABASE` 语句，需要先创建数据库。

### 3. 启动应用

启动 Spring Boot 应用，系统会自动执行数据库迁移（如果使用 Flyway）。

### 4. 访问系统初始化页面

打开浏览器访问系统（如 `http://localhost:8080`），系统会自动检测是否已初始化：

- **如果未初始化**: 自动跳转到初始化页面 (`/init`)
- **如果已初始化**: 跳转到登录页面 (`/login`)

### 5. 创建超级管理员

在初始化页面填写超级管理员信息：

- **用户名**: 3-50个字符，不能为空
- **邮箱**: 有效的邮箱地址
- **密码**: 8-50个字符，建议包含大小写字母、数字和特殊字符
- **确认密码**: 与密码一致

点击"创建超级管理员"按钮，系统会创建第一个超级管理员账号。

### 6. 登录系统

创建成功后，自动跳转到登录页面，使用刚才创建的超级管理员账号登录。

⚠️ **重要**: 请妥善保管超级管理员账号信息！

## 常见问题

### Q: 初始化页面无法访问怎么办？

A: 检查以下几点：
1. 应用是否正常启动
2. 数据库连接是否正常
3. 查看应用日志，定位具体错误
4. 确认路由配置是否正确

### Q: 如何创建新的超级管理员？

A: 使用现有的超级管理员账号登录，在用户管理页面创建新用户并设置为超级管理员角色。

### Q: 忘记超级管理员密码怎么办？

A: 可以通过以下方式重置：
1. 如果有其他超级管理员，请其他管理员重置密码
2. 如果没有其他管理员，需要直接在数据库中重置（需要数据库访问权限）

### Q: 为什么不能删除最后一个超级管理员？

A: 系统要求至少保留一个超级管理员，以确保系统始终有管理员可以管理。这是安全保护机制。

### Q: 初始化失败怎么办？

A: 检查以下几点：
1. 数据库连接配置是否正确
2. 数据库用户是否有足够权限
3. 表是否已存在（如果已存在，检查是否需要先删除）
4. 查看应用日志，定位具体错误
5. 检查表单验证是否通过（用户名、邮箱、密码格式）

### Q: 可以跳过初始化页面吗？

A: 不可以。系统要求必须通过初始化页面创建第一个超级管理员，这是安全设计，确保管理员信息由用户自己设置。
