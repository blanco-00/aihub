## 设计理念

### 核心原则
- **简洁高效**: 减少认知负担，提高操作效率
- **信息层次**: 重要信息突出，次要信息收敛
- **一致性**: 统一的视觉语言和交互模式
- **可扩展性**: 支持未来功能扩展
- **科技感**: 体现AI项目的科技属性，营造未来感和专业感

**布局结构**:

│  欢迎信息 + 快捷操作                    │
├──────────┬──────────┬───────────────────┤

├──────────┴──────────┴───────────────────┤
│  Token消耗趋势图 (7天/30天切换)         │

│  最近活动 / 快速访问                    │

**关键元素**:
- 数据卡片：大数字 + 趋势箭头 + 百分比
- 图表：使用 ECharts / Recharts

│  模型列表表格:                           │
│  - 模型名称                              │

│  - 状态 (可用/不可用)                   │

**模型配置表单**:
- 基本信息：名称、厂商、模型ID

## 交互设计

### 通用交互模式

1. **加载状态**
   - 骨架屏 (Skeleton)
   - 加载动画
   - 进度条

2. **反馈机制**
   - 成功：Toast提示 + 图标
   - 错误：错误提示 + 错误详情
   - 警告：警告提示

3. **确认操作**
   - 删除等危险操作需要二次确认
   - 使用Modal确认框

4. **表单验证**
   - 实时验证
   - 错误信息清晰
   - 必填项标记

### 动画效果

- 页面切换：淡入淡出
- 卡片悬停：轻微阴影变化，配合微妙的上升效果
- 按钮点击：按压效果，配合科技感的渐变
- 数据更新：数字滚动动画（countup效果）
- 加载动画：使用科技感的粒子效果或脉冲效果
- 数据可视化：图表支持平滑的动画过渡

## 科技感设计

### 设计目标
- **体现AI项目的科技属性**：通过视觉元素、配色、动画等营造未来感和专业感
- **保持现代感**：使用现代化的设计语言和交互模式
- **平衡美观与实用**：科技感应服务于功能，不应影响可用性

### 视觉元素
- **配色增强**：
  - 主色调使用科技蓝 `#1890ff`，体现科技感和专业性
  - 在关键区域使用蓝色渐变（如按钮、卡片背景）
  - 支持深色模式，增强科技感
  - 在数据可视化、状态指示等场景使用微妙的发光效果

- **几何图形**：
  - 使用简洁的几何图形作为装饰元素（如六边形、圆形、线条）
  - 在仪表盘等页面使用微妙的网格或点阵背景
  - 保持简洁，避免过度装饰

- **卡片设计**：
  - 使用卡片容器，配合阴影和圆角
  - 悬停时使用平滑的阴影变化和轻微上升效果
  - 关键数据使用等宽字体（monospace）显示

- **图标风格**：
  - 使用线性图标或填充图标，保持统一风格
  - 优先使用 Element Plus 图标库
  - 关键操作使用图标+文字的组合

### 交互增强
- **微交互**：
  - 按钮悬停、卡片悬停使用平滑的过渡动画
  - 数据更新使用滚动动画（countup效果）
  - 加载状态使用科技感的动画效果

- **数据可视化**：
  - 使用现代化的图表样式（ECharts、Recharts）
  - 支持动画效果和平滑过渡
  - 关键数据使用大数字+趋势指示的设计

### 实现建议
- 使用 CSS 渐变和阴影营造层次感
- 合理使用 `transform` 和 `transition` 实现流畅动画
- 数据展示使用等宽字体（monospace）增强科技感
- 关键操作按钮使用渐变背景和阴影效果

## 组件库

### Element Plus
本项目使用 **Element Plus** 作为 UI 组件库，原因如下：

- ✅ **Vue 3 官方推荐**：专为 Vue 3 设计的企业级组件库
- ✅ **组件丰富**：提供完整的后台管理系统所需组件
- ✅ **组件质量高**：经过大量项目验证，稳定可靠
- ✅ **中文文档完善**：便于团队协作和开发
- ✅ **设计风格统一**：符合企业级应用的设计规范
- ✅ **支持主题定制**：可以自定义主题色和样式
- ✅ **TypeScript 支持**：完整的类型定义
